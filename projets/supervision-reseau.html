Supervision Réseau — Centreon, GLPI, Wazuh

Plateforme intégrée de supervision réseau combinant monitoring d'infrastructure (Centreon), gestion d'incidents (GLPI) et corrélation de sécurité (Wazuh) pour une infrastructure critique.

Durée 2 mois (Oct-Nov 2024)
Équipe 3 personnes
Rôle Administrateur système & Intégrateur
Cadre TP Cybersécurité IUT

Contexte et Problématique

Dans le cadre d'un laboratoire de cybersécurité, nous devions mettre en place une solution complète de supervision réseau capable de :

  • Monitorer l'état de santé de l'infrastructure réseau (routeurs, switches, serveurs)
  • Gérer le cycle de vie complet des incidents et demandes de service
  • Détecter les anomalies de sécurité et corréler les événements
  • Fournir une vue consolidée pour les équipes opérationnelles et sécurité
  • Automatiser la création de tickets depuis les alertes de monitoring

Défi principal : Intégrer trois outils majeurs avec des logiques différentes tout en assurant une expérience utilisateur cohérente et une transmission fluide des informations.

Architecture de la Solution

Infrastructure virtualisée sur trois VMs Debian avec intégrations bidirectionnelles :

Centreon

Monitoring Infrastructure

• Supervision via SNMP, SSH, agents

• 50+ services monitorés

• Alerting et escalades

• Graphiques de performance

GLPI

Gestion d'Incidents

• ITSM complet (tickets, assets)

• Base de connaissances

• Workflow d'approbation

• SLA et reporting

Wazuh

SIEM & Security

• Détection d'intrusions

• Monitoring d'intégrité (FIM)

• Analyse de vulnérabilités

• Corrélation d'événements

Intégrations clés :

  • Centreon → GLPI : Création automatique de tickets depuis les alertes critiques
  • Wazuh → Centreon : Remontée des alertes de sécurité dans les dashboards
  • GLPI → Centreon : Consultation du statut des équipements depuis les tickets
  • Authentification centralisée via LDAP pour Single Sign-On

Mise en Œuvre Technique

1. Installation et Configuration

Déploiement sur VMs Debian 11 avec durcissement système :

  • Centreon : Installation via dépôts officiels, configuration MariaDB, PHP-FPM
  • GLPI : Déploiement LAMP stack, plugins (Fusion Inventory, Webhooks)
  • Wazuh : Installation manager + agents, configuration Elasticsearch/Kibana

2. Configuration SNMP

Monitoring réseau via SNMP v2c/v3 :

  • Configuration des équipements réseau (routeurs, switches)
  • Création de modèles de services Centreon pour différents vendors
  • Seuils d'alerting personnalisés par type d'équipement
  • Graphiques de bande passante et utilisation des interfaces

3. Intégration Centreon-GLPI

Automatisation de la gestion d'incidents :

  • Développement de scripts PHP pour l'API GLPI
  • Configuration des notifications Centreon avec webhook personnalisé
  • Enrichissement automatique des tickets (logs, graphiques, contexte)
  • Mise à jour bidirectionnelle du statut (résolution dans GLPI → ACK dans Centreon)

4. Sécurisation Wazuh

Déploiement des agents et règles de détection :

  • Installation agents Wazuh sur tous les serveurs monitorés
  • Configuration FIM sur répertoires critiques (/etc, /bin, /root)
  • Création de règles personnalisées pour événements spécifiques
  • Intégration avec VirusTotal pour analyse de malwares

Défis Techniques Rencontrés

⚠️ Authentification MySQL

Problème : Erreur "Access denied for user 'centreon'@'localhost'"

Solution : Reconfiguration du plugin auth_socket MySQL, migration vers mysql_native_password, révision des privilèges.

Connectivité SNMP

Problème : Timeouts lors du polling de certains équipements

Solution : Ajustement des paramètres SNMP (timeout, retry), vérification firewall, tests de performance réseau.

Agents Wazuh

Problème : Agents déconnectés après redémarrage des VMs

Solution : Configuration du service systemd pour auto-restart, ajustement des timeouts de connexion manager.

API GLPI

Problème : Rate limiting lors de la création massive de tickets

Solution : Implémentation d'un système de queue avec retry exponentiel, batch processing.

Retour d'Expérience

Ce projet m'a permis de comprendre l'importance d'une approche holistique de la supervision :

  • Visibilité complète : La corrélation entre monitoring infrastructure, gestion d'incidents et sécurité est essentielle pour détecter et résoudre rapidement les problèmes
  • Automatisation : La création automatique de tickets depuis les alertes réduit significativement le temps de réaction et évite la perte d'informations
  • Standardisation : L'utilisation de modèles et de procédures standardisées facilite l'exploitation et réduit les erreurs
  • Persistance : Le troubleshooting d'intégrations complexes nécessite patience, méthode et documentation rigoureuse
